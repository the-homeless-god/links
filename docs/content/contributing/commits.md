# Conventional Commits

Проект использует [Conventional Commits](https://www.conventionalcommits.org/) для автоматической генерации changelog и определения версий релизов.

## Формат коммита

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Типы коммитов

- **feat**: Новая функциональность
- **fix**: Исправление бага
- **docs**: Изменения в документации
- **style**: Изменения форматирования (не влияют на код)
- **refactor**: Рефакторинг кода
- **perf**: Улучшение производительности
- **test**: Добавление или изменение тестов
- **build**: Изменения в системе сборки
- **ci**: Изменения в CI конфигурации
- **chore**: Другие изменения (обновление зависимостей и т.д.)

### Примеры

#### Новая функциональность (minor версия)
```
feat(api): добавлена поддержка публичных ссылок

Теперь пользователи могут создавать публичные ссылки,
доступные без авторизации по пути /u/:name
```

#### Исправление бага (patch версия)
```
fix(auth): исправлена проверка токена гостя

Токен гостя теперь корректно проверяется при каждом запросе
```

#### Критическое изменение (major версия)
```
feat(api)!: изменен формат ответа API

BREAKING CHANGE: поле 'url' переименовано в 'target_url'
в ответах API для согласованности
```

#### Документация (не создает релиз)
```
docs: обновлена документация по установке

Добавлены инструкции для Windows
```

#### Рефакторинг (patch версия)
```
refactor(repo): упрощена логика сохранения ссылок

Удален избыточный код проверки дубликатов
```

## Автоматическая генерация релизов

При коммите в ветку `master` или `main`:

- **feat**: Создается **minor** релиз (1.0.0 → 1.1.0)
- **fix**: Создается **patch** релиз (1.0.0 → 1.0.1)
- **BREAKING CHANGE**: Создается **major** релиз (1.0.0 → 2.0.0)
- **docs**, **style**, **chore**, **test**, **ci**, **build**: Релиз не создается
