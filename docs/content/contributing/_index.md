# Участие в проекте

Спасибо за интерес к проекту Links API! Мы рады вашему участию.

## Как внести вклад

1. Форкните репозиторий
2. Создайте ветку для вашей функциональности (`git checkout -b feature/amazing-feature`)
3. Внесите изменения
4. Убедитесь, что код проходит все проверки
5. Закоммитьте изменения (`git commit -m 'Add some amazing feature'`)
6. Отправьте ветку (`git push origin feature/amazing-feature`)
7. Откройте Pull Request

## Стандарты кода

### Backend (Elixir)

- Используйте `mix format` для форматирования
- Запустите `mix credo` для проверки стиля
- Запустите `mix dialyzer` для проверки типов
- Напишите тесты для новой функциональности

### Extension (TypeScript)

- Используйте `npm run format` для форматирования
- Запустите `npm run lint` для проверки стиля
- Запустите `npm run check-types` для проверки типов
- Напишите тесты для новой функциональности

## Тестирование

Перед отправкой PR убедитесь, что:

- Все тесты проходят (`mix test` или `npm test`)
- Код отформатирован
- Нет ошибок линтера
- CI проверки проходят успешно

## Сообщение об ошибках

При сообщении об ошибке укажите:

- Версию системы
- Шаги для воспроизведения
- Ожидаемое поведение
- Фактическое поведение
- Логи (если применимо)

## Вопросы

Если у вас есть вопросы, откройте Issue в репозитории.
